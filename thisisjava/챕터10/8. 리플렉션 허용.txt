은닉된 패키지는 기본적으로 다른 모듈에 의해 리플렉션을 허용하지 않습니다.
리플렉션이란 실행 도중에 타입(클래스, 인터페이스 등)을 검사하고 구성 멤버를 조사하는 것을 말합니다.

모듈 전체를 리플렉션 허용
open module 모듈명 {
	...
}

지정된 패키지에 대해 리플렉션 허용
module 모듈명 {
	...
	opens 패키지1;
	opens 패키지2;
}

지정된 패키지에 대해 특정 외부 모듈에서만 리플렉션 허용
module 모듈명 {
	...
	opens 패키지1 to 외부모듈명, ...
	opens 패키지2 to 외부모듈명;
}

export된 패키지는 언제든지 리플렉션이 가능해서 opens로 지정할 필요가 없습니다.
